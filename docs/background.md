# ChatGroup - 项目背景

## 一、为什么要做这个项目

### 现有聊天软件的痛点

在日常使用微信、QQ、Telegram 等聊天软件与朋友交流的过程中，我们经常会遇到这样的问题：

1. **重要信息被淹没**：长时间的对话中，很多有价值的内容（约定的事项、讨论的结论、推荐的东西）混杂在大量闲聊中，事后很难找回。

2. **缺少对话总结能力**：现有聊天软件没有对聊天内容进行自动或半自动总结的功能。你想回忆"上次我们讨论了什么"，只能手动翻聊天记录。

3. **搜索功能有限**：即使用关键词搜索，也只能找到包含该关键词的消息，无法理解对话的上下文和语义。

4. **信息管理断层**：聊天中讨论的待办事项、做出的决定，需要手动复制到其他工具（备忘录、Todo App）中，这个过程容易遗漏。

### 解决思路

做一款内置 AI 总结能力的聊天软件，通过特定的唤醒词（如 `@ai`）触发 AI 对聊天记录进行智能总结，自动提取：
- 讨论的主题
- 达成的共识或决定
- 待办事项
- 关键信息（时间、地点、数字等）

这样，对话中的有价值信息就不会被遗忘。

## 二、项目定位

- **使用规模**：个人项目，暂时仅供我和朋友两个人使用
- **项目性质**：实验性项目，重在学习和实践，同时解决实际痛点
- **核心特色**：AI 驱动的对话总结功能

## 三、核心功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 用户注册与登录 | 基础的账号系统，JWT 鉴权 | P0 |
| 实时聊天 | 基于 WebSocket 的即时通信 | P0 |
| 聊天记录持久化 | 消息存储到数据库，支持历史记录查看 | P0 |
| 好友系统 | 好友添加、好友列表、在线状态 | P0 |
| AI 总结 | 通过唤醒词触发，AI 总结聊天内容 | P0 |
| 消息已读状态 | 显示消息是否被对方阅读 | P1 |
| "正在输入"提示 | 对方打字时显示状态提示 | P1 |
| 消息搜索 | 搜索历史聊天记录 | P2 |
| 总结历史 | 查看和管理过往的 AI 总结记录 | P2 |

## 四、AI 唤醒词设计

预设的唤醒词及对应功能：

| 唤醒词 | 功能 | 示例 |
|--------|------|------|
| `@ai 总结` | 总结最近的对话 | `@ai 总结今天的聊天` |
| `@ai 待办` | 提取对话中的待办事项 | `@ai 待办` |
| `@ai 帮我记一下` | 记录特定信息 | `@ai 帮我记一下 周六下午三点在星巴克见面` |
| `@ai 问` | 基于聊天上下文回答问题 | `@ai 问 我们上次说的那个电影叫什么` |

唤醒词可自定义，支持 `@ai`、`/ai` 等多种触发方式。

## 五、目标用户画像

- 我自己和朋友
- 经常通过文字聊天讨论事情
- 有回顾聊天内容的需求
- 希望对话中的有价值信息能被自动整理

## 六、与现有产品的对比

| 特性 | 微信 | Telegram | Slack | **ChatGroup** |
|------|------|----------|-------|---------------|
| 实时聊天 | Yes | Yes | Yes | Yes |
| 聊天记录搜索 | 关键词 | 关键词 | 关键词 | 关键词 + AI 语义 |
| AI 总结 | No | No | 第三方插件 | **内置核心功能** |
| 待办提取 | No | No | 第三方插件 | **内置** |
| 开源可控 | No | 客户端开源 | No | **完全开源** |
| 数据隐私 | 存储在腾讯服务器 | 云端 | 云端 | **自建服务器，完全可控** |

## 七、开发阶段规划

### 阶段一：基础功能
- 用户注册、登录
- 实时聊天（文本消息）
- 好友系统
- 聊天记录存储与查看

### 阶段二：AI 能力
- 唤醒词检测
- 对话总结功能
- 总结结果展示与存储

### 阶段三：体验优化
- 消息已读状态
- "正在输入"提示
- 断线重连优化
- UI 打磨

### 阶段四：扩展功能（可选）
- 消息搜索
- 总结历史管理
- 图片/文件发送
- 移动端适配
